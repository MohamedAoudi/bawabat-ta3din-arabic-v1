## SQL Generation Rules
- Tables available: countries, mineral_production, mineral_trade, trade_partners, arab_production, world_production, trade_world, partner_trade
- Always JOIN countries table to get multilingual names (name_ar, name_en, name_fr)
- Always JOIN mineral_production table to get mineral names (mineral_name_ar, mineral_name_en, mineral_name_fr)
- For production queries: use arab_production and world_production tables
- For trade queries: use trade_world and partner_trade tables
- NEVER use '=' to filter a country, mineral, or partner name. Names are stored as
  official long forms (e.g. 'Kingdom of Morocco', 'Phosphate rock'). ALWAYS match
  with case-insensitive ILIKE on a distinctive token, e.g. c.name_en ILIKE '%Morocco%'
  and mp.mineral_name_en ILIKE '%phosphate%'. Use the GROUND TRUTH country list in the schema context.
- type_trade values are SINGULAR: 'export' and 'import'. Match them with ILIKE, e.g. type_trade ILIKE 'export'
- Use production_value_base for production aggregations (normalized values)
- Use value_usd for trade aggregations
- Always include appropriate LIMIT clause (default: 100 for lists)
- Return ONLY the SQL query, no explanations or markdown fences
- If the question cannot be answered with the available schema, return: NO_SQL
